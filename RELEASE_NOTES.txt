ClevoLEDKeyboardControl v2.4.0

稳定性与自动化：
- 自动化会跳过无效、字段缺失或预设已删除的规则，不再阻塞后续音乐程序、灯效程序和时间计划。
- 配置损坏时保留带时间戳的故障文件，并优先恢复最近一次有效备份。
- 停用规则或关闭自动化后，不再错误启用键盘钩子和通知监视器。
- 通知检测回退轮询缩短至约 1 秒，并保留通知冷却策略。

安全通信与配置：
- 托盘与服务改用带 ACL、协议版本、1 MB 消息限制及活动用户会话校验的命名管道通信。
- 服务配置目录收紧为普通用户只读；配置写入由服务原子完成。
- 更新检查状态迁移至当前用户 LocalAppData。
- 升级前额外备份 settings.json.pre-security-v2.4.bak；管道不可用时设置保持只读。

场景与诊断：
- 新增场景模拟器，可模拟时间、前台程序、播放程序、音量和空闲状态，不改变实际灯光。
- 新增规则健康检查，提示预设丢失、进程为空、规则覆盖和低置信度媒体绑定。
- 新增配置导入、导出和最近有效备份恢复；无效导入不会覆盖当前设置。
- 诊断页显示安全管道、音频程序、媒体会话、配置恢复及捕获回退状态。

音乐与封面：
- 媒体绑定优先使用精确会话 ID；自动匹配存在多个候选时回退音乐预设颜色，避免误用其他播放器封面。
- 封面缓存键加入专辑信息和图片内容哈希，播放器更新封面后会重新提色。
- 音乐页可禁止系统混音频段回退；普通响应继续使用目标程序自己的音频会话电平。

开发与更新：
- 增加标准 ColorfulLedKeyboard.sln 和 global.json，固定兼容的 .NET 8 SDK。
- 自动测试扩展至 100 项。
- 为后续升级包签名校验和应用内更新预留接口；当前仍从 GitHub Release 手动下载安装。

托盘与更新提醒（合并自 2.3.6）：
- 托盘实时显示模式、播放器、歌曲、场景和封面状态，并可快捷控制规则、预设、事件反馈与亮度。
- 启动和运行期间自动检查更新，每个新版本只主动提示一次。
- 更新检查使用 GitHub releases/latest 网页跳转，不消耗 GitHub REST API 匿名限额。
