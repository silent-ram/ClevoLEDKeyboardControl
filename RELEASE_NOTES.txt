ClevoLEDKeyboardControl v2.3.6

更新检查修复：
- 更新检查改用 GitHub releases/latest 网页跳转识别版本，不再消耗 GitHub REST API 匿名请求配额。
- 修复共享公网 IP 或频繁检查后出现 403 rate limit exceeded 的问题。
- 检查失败提示改为简明中文，不再直接显示英文网络异常。

托盘快捷控制：
- 托盘顶部实时显示当前模式、播放器、歌曲、命中场景和封面状态。
- 可快捷启停场景自动化及单条音乐程序、灯效程序和时间计划规则。
- 可切换基础灯效/音乐预设、绑定当前有声程序、选择封面颜色来源。
- 可快捷开关敲字与通知反馈，并调整普通亮度或音乐峰值亮度。

更新提醒：
- 启动后自动检查，运行期间定时检查更新。
- 每个新版本自动弹出一次“立即下载/稍后”提示，避免重复打扰。
- 稍后处理时，托盘菜单和悬停提示持续显示可用版本及下载入口。

界面改进：
- 音乐页显示媒体会话自动匹配的实时结果、命中的会话 ID、播放/切歌状态和封面颜色数量。
- 自动匹配失败时列出当前可用媒体会话，便于切换为手动选择。
- 修正“自动匹配播放器”被错误显示为“未匹配媒体会话”的提示。

封面颜色优化：
- 切歌、短暂停止或新封面尚未加载时保留上一组有效颜色，避免瞬间回退音乐预设。
- 封面会话刷新缩短至约 500ms，并按歌曲缓存最多 128 组配色。
- 音乐设置页实时显示当前歌曲、实际颜色值和配色色条。
- 封面主色模式扩大明暗律动范围，并增强过暗、低饱和封面色的键盘可见度。

关键修复：
- 音频会话和 PID 改由用户会话中的托盘程序采集，解决 LocalSystem 服务处于 Session 0、无法看到 QQ 音乐等用户播放器的问题。
- 托盘以原子状态文件把程序 PID、实时电平和播放状态交给服务，场景自动化关闭时也持续可用。

修复：
- 修复关闭场景自动化或使用静态灯效时，QQ 音乐等播放器不会出现在有声程序列表的问题。
- 音乐页新增独立播放器 PID 绑定；无需启用场景自动化即可使用指定程序音频和歌曲封面颜色。
- 绑定窗口同时显示已检测到的音频会话和具有主窗口的运行程序，支持播放器主进程带音频子进程的情况。
- 音频会话检测覆盖所有活动输出设备，不再只检查默认多媒体设备。

新增：
- 场景自动化拆分为音乐程序、灯效程序和时间计划。
- 音乐程序按实际音频会话电平匹配，支持前台播放器优先和后台规则顺序。
- 音乐页面可绑定当前有声程序，保存稳定进程身份并自动跟踪新 PID。
- 支持从 Windows 媒体会话提取歌曲封面主色或多色配色。
- 音乐模式支持打字亮度反馈；打字和通知可按程序继承、开启或关闭。
- 支持前台灯效程序在音乐运行时叠加亮度和事件策略。

兼容说明：
- 播放器未向 Windows 提供封面时自动使用音乐预设颜色。
- 普通音乐响应使用目标程序自己的会话电平；自适应频段分析仍会回退系统混音。
- 配置升级前会备份为 settings.json.pre-automation-v2.bak。
v2.3.7
- 自动化会跳过无效或预设丢失的规则，不再阻塞后续音乐、灯效和时间规则。
- 配置损坏时保留故障文件并优先恢复最近有效备份，诊断页显示恢复结果。
- 停用规则或关闭自动化后不再错误启用键盘与通知监视器。
- 改进播放器诊断、媒体会话匹配和封面缓存，通知检测延迟缩短至约 1 秒。
- 增加标准解决方案文件并固定 .NET 8 SDK，完善回归测试。
v2.4.0
- 托盘与服务使用带 ACL、协议版本、消息大小限制和客户端会话校验的命名管道通信。
- 服务配置目录改为普通用户只读，更新状态迁移到用户 LocalAppData；管道不可用时设置保持只读。
- 增加场景模拟器、规则健康检查、配置导入导出与最近有效备份恢复。
- 音乐页可禁止系统混音频段回退，运行状态明确显示实际捕获/回退方式。
- 为后续签名校验与应用内更新预留验证接口，当前仍通过 GitHub Release 手动下载。
